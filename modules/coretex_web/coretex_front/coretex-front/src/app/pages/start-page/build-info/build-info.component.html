Build process separated by two parts build and DB initialization

Build:

Building process include a several tasks that dedicated to generate additional java classes that provide an access to database also during build process will be generated script for db initialization.

To run build you need to perform simple Gradle 'build' instruction in the project directory

Example:
./gradlew build

To clean up all generated Items, SQL scripts, etc. You need to perform 'clean' instruction

./gradlew clean

You also can combine this two instruction:

./gradlew clean build
Commands:
Command Description Example
clean Used to remove generated java classes and sql scripts ./gradlew clean
generate Used to generate java classes and sql script based on information collected from *.struct files ./gradlew generate
build Start the build process and run 'generate' command automatically before java compilation process. Also download required libs for each module and place it to 'libs' directory ./gradlew build

DB initialization:

Db initialization is the second part of build process dedicated to create database, tables and essential information for ORM

Example:
./gradlew initialize

If database already initialized but data structure has been changed you need to perform data base update. For this purposes exist 'updatesystem' task

Example:
./gradlew updatesystem
Commands:
Command Description Example
initialize Create database defined in coretex.properties if not exist. Create if not exist and execute sql initial script ./gradlew initialize
updatesystem Used to migrate initialized DB to new structure. Automatically create and execute sql commands that required to cover diff in table structure and meta data ./gradlew updatesystem
