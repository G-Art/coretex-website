.struct files contain a script that allows describing Java items, database table structure and relations. Struct file contain three elements: relations, items, enums

relations - contain definition for principal relation between items and represent DB table relation as one-to-one, one-to-many and many-to-many.
Relations used for declaring bidirectional relation.

items - contain definition for item that represent object-relational mapping and allow system generate java class and DB structure as table and metadata

enums - contain definition for enums that represent enums class and allow to use it as type for DB table column

Relations declaration

There are two ways to create a relation between items:

  1. One direction, is just create a field with a type of another item example: (firstExampleOneDirection(FirstExample))
  2. Define the relation in the relations section. This approach has a few advantages.
  • Allows creating bidirectional relation.
  • Allows crete 1 <=> many relation
  • Allows crete many <=> many relation

Declaration for new relation contains the next parts:

  • Relation name example: (ManyToManyBidirectionalRelation)
  • Source and tasget type declaration example: (source: FirstExample, target: SecondExample).
    Attributes source and target is mandatory
  • Declaration source(mtmSecondExampleMany) in relation body means that for source type (FirstExample) will be created field with name mtmSecondExampleMany and type defined to target.
  • Optional attributes for source and target declaration is "containerType", "associated".
Parameter Value Description
containerType Set or List (Optional) Used for many items relation and specify that collection type will use for this field
associated true or false (Optional) All items related to this field will be removed once remove owner. default: false

Item declaration

Item is a declaration of a Java class that represents a table with relations as a java object

Item has the following parameters:

Parameter Value Description
extend (Declared item name) (Optional) Using to extend existing type same as java extends. By default extends Generic item declared in core module
table true or false (Optional) By default "true" if true type will get dedicated table, if "false" all specified fields will store in closest parent item with dedicated table.
abstract true or false (Optional) By default "false" affect only java class and make item abstract if it's required.
enhance module name (Optional) Used to add additional attributes to already exist Item example: (enhance: 'core') just need to define module name that declares desired item.
